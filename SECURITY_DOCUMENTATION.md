## 1. Authentication Implementation

### 1.1 Single-Factor Authentication (SFA)

**Implementation Method**: Username/Password Authentication

#### Password Storage Security

```javascript
// Hashing with bcrypt (10 rounds)
const salt = await bcrypt.genSalt(10);
const hashedPassword = await bcrypt.hash(password, salt);

// Storage format
user = {
  username: "john_doe",
  password: "$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17tlaq",
  salt: "$2a$10$N9qo8uLOickgx2ZMRZoMye", // Auto-generated by bcrypt
  email: "john@example.com",
};
```

#### Verification Process

```javascript
// Password comparison
const validPassword = await bcrypt.compare(inputPassword, storedHash);

if (validPassword) {
  // Generate JWT token
  const token = jwt.sign(
    { id: user.id, username: user.username, role: user.role },
    SECRET_KEY,
    { expiresIn: "24h" },
  );
}
```

**Security Benefits:**

- No plaintext password storage
- Computational cost prevents brute force
- Unique salt per password
- Protection against rainbow tables

## 2. Authorization & Access Control

### 2.1 Access Control Matrix (ACL)

**Model**: Role-Based Access Control (RBAC)

#### Complete ACL Table

| Subject (Role) | Documents           | Requests            | Reports             |
| -------------- | ------------------- | ------------------- | ------------------- |
| **Admin**      | Read, Write, Delete | Read, Write, Delete | Read, Write, Delete |
| **Manager**    | Read, Write         | Read, Write         | Read                |
| **Employee**   | Read                | Read, Write         | None                |

#### Code Implementation

```javascript
const accessControlMatrix = {
  admin: {
    documents: ["read", "write", "delete"],
    requests: ["read", "write", "delete"],
    reports: ["read", "write", "delete"],
  },
  manager: {
    documents: ["read", "write"],
    requests: ["read", "write"],
    reports: ["read"],
  },
  employee: {
    documents: ["read"],
    requests: ["read", "write"],
    reports: ["none"],
  },
};
```

### 2.2 Policy Definition & Justification

#### Admin Role

**Permissions**: Full access (Read, Write, Delete) on all objects

**Justification**:

- System administrators need complete control for maintenance
- Required for user management and system configuration
- Oversight of all organizational documents and requests
- Audit trail management and security monitoring
- Emergency access for critical situations

**Use Cases**:

- System configuration and maintenance
- User role assignment and management
- Security policy enforcement
- Backup and recovery operations

#### Manager Role

**Permissions**:

- Documents: Read, Write
- Requests: Read, Write
- Reports: Read only

**Justification**:

- Managers oversee team document workflows
- Need to approve/deny employee access requests
- Can create and modify departmental documents
- Read-only reports prevent accidental deletion of analytics
- No delete permission prevents accidental data loss
- Balance between functionality and safety

**Use Cases**:

- Approve employee access requests
- Review team performance reports
- Create departmental documentation
- Manage document access policies

#### Employee Role

**Permissions**:

- Documents: Read only
- Requests: Read, Write
- Reports: No access

**Justification**:

- Employees consume documents for daily tasks
- Can request access to restricted documents
- Prevention of unauthorized document modification
- No access to sensitive organizational reports
- Aligns with principle of least privilege
- Reduces insider threat risk

**Use Cases**:

- Access approved documents
- Submit access requests for new documents
- View own request history
- Daily operational tasks

### 2.3 Programmatic Enforcement

#### Backend Authorization Middleware

```javascript
function authenticateToken(req, res, next) {
  const token = req.headers["authorization"]?.split(" ")[1];

  if (!token) {
    return res.status(401).json({ message: "Unauthorized" });
  }

  jwt.verify(token, SECRET_KEY, (err, user) => {
    if (err) return res.status(403).json({ message: "Forbidden" });
    req.user = user; // Contains role information
    next();
  });
}
```

#### Permission Checking

```javascript
// Example: Document deletion
app.delete("/api/documents/:id", authenticateToken, (req, res) => {
  const doc = db.documents.find((d) => d.id === req.params.id);

  // Check permissions
  const permissions = accessControlMatrix[req.user.role]?.documents || [];
  const canDelete = permissions.includes("delete");
  const isOwner = doc.owner === req.user.username;

  if (!canDelete || !isOwner) {
    return res.status(403).json({ message: "Permission denied" });
  }

  // Proceed with deletion
});
```

#### Frontend Permission Enforcement

```javascript
function canAccessDocument(doc) {
  const role = currentUser.role;
  const permissions = accessControlMatrix[role]?.documents || [];
  return (
    permissions.includes("read") &&
    (doc.owner === currentUser.username || doc.approved)
  );
}

// Conditional UI rendering
if (canDeleteDocument(doc)) {
  showDeleteButton();
}
```

## 3. Encryption Implementation

### 3.1 Key Generation & Exchange

**Algorithm**: AES-256-GCM (Galois/Counter Mode)

#### Key Generation

```javascript
async function generateEncryptionKey() {
  // Generate 256-bit AES key
  const key = await crypto.subtle.generateKey(
    { name: "AES-GCM", length: 256 },
    true, // extractable
    ["encrypt", "decrypt"],
  );

  // Export key to raw format
  const exported = await crypto.subtle.exportKey("raw", key);

  // Encode as Base64 for storage/transmission
  return btoa(String.fromCharCode(...new Uint8Array(exported)));
}

// Example output: "xK7mqz9p2xK7mqz9p2xK7mqz9p2xK7mqz9=="
```

**Key Properties**:

- 256-bit strength (industry standard)
- Cryptographically random generation
- Unique per document
- Secure export/import mechanism

### 3.2 Encryption Process

#### Step-by-Step Encryption

```javascript
async function encryptText(plaintext, keyBase64) {
  // 1. Import key from Base64
  const keyData = Uint8Array.from(atob(keyBase64), (c) => c.charCodeAt(0));
  const key = await crypto.subtle.importKey(
    "raw",
    keyData,
    { name: "AES-GCM" },
    false,
    ["encrypt"],
  );

  // 2. Generate random IV (12 bytes for GCM)
  const iv = crypto.getRandomValues(new Uint8Array(12));

  // 3. Encrypt data
  const encoder = new TextEncoder();
  const encrypted = await crypto.subtle.encrypt(
    { name: "AES-GCM", iv: iv },
    key,
    encoder.encode(plaintext),
  );

  // 4. Combine IV + ciphertext
  const combined = new Uint8Array(iv.length + encrypted.byteLength);
  combined.set(iv);
  combined.set(new Uint8Array(encrypted), iv.length);

  // 5. Encode to Base64
  return btoa(String.fromCharCode(...combined));
}
```

**Security Features**:

- Authentication tag (GCM mode)
- Random IV per encryption
- IV included with ciphertext
- No IV reuse (prevents attacks)

### 3.3 Decryption Process

```javascript
async function decryptText(encryptedBase64, keyBase64) {
  // 1. Import decryption key
  const keyData = Uint8Array.from(atob(keyBase64), (c) => c.charCodeAt(0));
  const key = await crypto.subtle.importKey(
    "raw",
    keyData,
    { name: "AES-GCM" },
    false,
    ["decrypt"],
  );

  // 2. Decode Base64 and extract IV
  const combined = Uint8Array.from(atob(encryptedBase64), (c) =>
    c.charCodeAt(0),
  );
  const iv = combined.slice(0, 12);
  const ciphertext = combined.slice(12);

  // 3. Decrypt
  const decrypted = await crypto.subtle.decrypt(
    { name: "AES-GCM", iv: iv },
    key,
    ciphertext,
  );

  // 4. Decode to text
  const decoder = new TextDecoder();
  return decoder.decode(decrypted);
}
```

**Error Handling**:

- Authentication tag verification
- Automatic tamper detection
- Invalid key detection
- Corruption detection

## 4. Hashing & Digital Signatures

### 4.1 Password Hashing with Salt

**Algorithm**: bcrypt (Blowfish cipher)

#### Implementation

```javascript
// Registration
const salt = await bcrypt.genSalt(10); // 2^10 iterations
const hashedPassword = await bcrypt.hash(password, salt);

// Storage
{
    username: "alice",
    password: "$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17tlaq",
    // Salt is embedded in the hash (first 29 characters)
}

// Verification
const isValid = await bcrypt.compare(inputPassword, storedHash);
```

**Security Properties**:

- **Slow by design**: Prevents brute force (adjustable cost factor)
- **Unique salt per password**: Prevents rainbow table attacks
- **Adaptive**: Can increase cost factor over time
- **Industry standard**: Used by major platforms

#### Custom SHA-256 Hashing

```javascript
// Generate salt
function generateSalt(length = 16) {
  const array = new Uint8Array(length);
  crypto.getRandomValues(array);
  return Array.from(array, (byte) => byte.toString(16).padStart(2, "0")).join(
    "",
  );
}

// Hash with salt
async function hashWithSalt(text, salt) {
  const encoder = new TextEncoder();
  const data = encoder.encode(text + salt);
  const hashBuffer = await crypto.subtle.digest("SHA-256", data);
  const hashArray = Array.from(new Uint8Array(hashBuffer));
  return hashArray.map((b) => b.toString(16).padStart(2, "0")).join("");
}

// Example
const salt = generateSalt(); // "a3f2c8d9e1b4..."
const hash = await hashWithSalt("secret", salt);
// "5e884898da28047151d0e56f8dc629..."
```

### 4.2 Digital Signatures

**Algorithm**: HMAC-SHA256 (Hash-based Message Authentication Code)

#### Signature Generation

```javascript
async function generateSignature(text, secret = "SECRET_KEY_2026") {
  // 1. Create HMAC key
  const encoder = new TextEncoder();
  const keyData = encoder.encode(secret);
  const key = await crypto.subtle.importKey(
    "raw",
    keyData,
    { name: "HMAC", hash: "SHA-256" },
    false,
    ["sign"],
  );

  // 2. Generate signature
  const signature = await crypto.subtle.sign("HMAC", key, encoder.encode(text));

  // 3. Convert to hex string
  const hashArray = Array.from(new Uint8Array(signature));
  return hashArray.map((b) => b.toString(16).padStart(2, "0")).join("");
}

// Example output: "3a7f5e2bc8d94a1e6f..."
```

#### Signature Verification

```javascript
async function verifySignature(text, signature, secret = "SECRET_KEY_2026") {
  const newSignature = await generateSignature(text, secret);
  return newSignature === signature; // Constant-time comparison
}

// Usage
const doc = "Important contract v1.0";
const signature = await generateSignature(doc);

// Later verification
const isValid = await verifySignature(doc, signature);
if (isValid) {
  console.log("✅ Document is authentic and unmodified");
} else {
  console.log("❌ Document has been tampered with!");
}
```

**Security Properties**:

- **Integrity**: Detects any modification to data
- **Authenticity**: Verifies sender identity
- **Non-repudiation**: Cannot deny creating signature
- **Key-based**: Requires secret key to generate

### 4.3 Document Integrity System

```javascript
// Complete workflow
async function uploadDocument(content) {
  // 1. Encrypt content
  const key = await generateEncryptionKey();
  const encrypted = await encryptText(content, key);

  // 2. Generate signature of ORIGINAL content
  const signature = await generateSignature(content);

  // 3. Store all components
  return {
    encryptedContent: encrypted,
    encryptionKey: key,
    signature: signature,
  };
}

// Verification on retrieval
async function verifyAndDecrypt(doc) {
  // 1. Decrypt content
  const decrypted = await decryptText(doc.encryptedContent, doc.encryptionKey);

  // 2. Verify signature
  const isValid = await verifySignature(decrypted, doc.signature);

  if (!isValid) {
    throw new Error("Document integrity check failed!");
  }

  return decrypted;
}
```

## 5. Encoding Techniques

### 5.1 Base64 Encoding/Decoding

**Purpose**: Binary-to-text encoding for data transmission

#### Implementation

```javascript
// Encode text to Base64
function base64Encode(text) {
  return btoa(unescape(encodeURIComponent(text)));
}

// Decode Base64 to text
function base64Decode(encoded) {
  return decodeURIComponent(escape(atob(encoded)));
}

// Example
const original = "Hello, 世界! 🌍";
const encoded = base64Encode(original);
// "SGVsbG8sIOS4lueVjCEg8J+MjQ=="

const decoded = base64Decode(encoded);
// "Hello, 世界! 🌍"
```

**Use Cases**:

- Encoding encryption keys for transmission
- Storing binary data in JSON
- Encoding encrypted content
- URL-safe data representation

## 6. Security Analysis

### 6.1 Security Levels

#### 1. Public (Level 0)

**Characteristics**:

- No encryption required
- Accessible to all authenticated users
- Basic access logging

**Examples**: Company announcements, public policies

**Risk Level**: Low

#### 2. Internal (Level 1)

**Characteristics**:

- Basic encryption (AES-128 acceptable)
- Organization-wide access
- Access control by role

**Examples**: Internal memos, process documents

**Risk Level**: Medium

#### 3. Confidential (Level 2)

**Characteristics**:

- Strong encryption (AES-256 required)
- Role-based access control
- Audit logging required
- Digital signatures recommended

**Examples**: Financial reports, HR records

**Risk Level**: High

#### 4. Restricted (Level 3)

**Characteristics**:

- Maximum encryption (AES-256-GCM)
- Admin-only access or explicit approval
- Comprehensive audit trail
- Mandatory digital signatures
- Multi-factor authentication required
- Data loss prevention (DLP)

**Examples**: Trade secrets, legal documents, executive communications

**Risk Level**: Critical

### 6.2 Risk Assessment

#### Authentication Risks

| Risk                     | Severity | Mitigation                                |
| ------------------------ | -------- | ----------------------------------------- |
| Brute Force Attack       | High     | Rate limiting, Account lockout, 2FA       |
| Credential Stuffing      | High     | Password complexity, Breach monitoring    |
| Session Hijacking        | Medium   | HTTPS, HTTPOnly cookies, Token expiration |
| Password Database Breach | High     | bcrypt hashing, Unique salts              |

#### Encryption Risks

| Risk              | Severity | Mitigation                             |
| ----------------- | -------- | -------------------------------------- |
| Weak Key          | Critical | 256-bit keys, Crypto-random generation |

| Risk                  | Severity | Mitigation                                    |

### 6.3 Possible Attacks & Defenses

#### 1. Brute Force Attack

**Description**: Automated attempts to guess passwords

**Mitigation**:

```javascript
// Rate limiting example
const loginAttempts = new Map();

function checkRateLimit(username) {
  const attempts = loginAttempts.get(username) || 0;
  if (attempts >= 5) {
    throw new Error("Too many login attempts. Try again in 15 minutes.");
  }
  loginAttempts.set(username, attempts + 1);
}

// Reset after successful login
function resetAttempts(username) {
  loginAttempts.delete(username);
}
```

#### 2. SQL Injection

**Description**: Malicious SQL code injection

**Mitigation**:

```javascript
// Use parameterized queries
db.query("SELECT * FROM users WHERE username = ?", [username]);

// Input validation
function sanitizeInput(input) {
  return input.replace(/[^\w\s@.-]/gi, "");
}
```

#### 3. Cross-Site Scripting (XSS)

**Description**: Injecting malicious scripts

**Mitigation**:

```javascript
// Sanitize output
function escapeHTML(text) {
  const div = document.createElement("div");
  div.textContent = text;
  return div.innerHTML;
}

// Content Security Policy
app.use((req, res, next) => {
  res.setHeader(
    "Content-Security-Policy",
    "default-src 'self'; script-src 'self'",
  );
  next();
});
```

#### 4. Man-in-the-Middle (MITM)

**Description**: Intercepting communication

**Mitigation**:

- Use HTTPS/TLS for all communications
- Implement certificate pinning
- Use HSTS (HTTP Strict Transport Security)

```javascript
// HSTS header
app.use((req, res, next) => {
  res.setHeader(
    "Strict-Transport-Security",
    "max-age=31536000; includeSubDomains",
  );
  next();
});
```

#### 5. Session Hijacking

**Description**: Stealing session tokens

**Mitigation**:

```javascript
// Secure JWT configuration
const token = jwt.sign(payload, SECRET, {
  expiresIn: "24h",
  issuer: "docaccess.com",
  audience: "docaccess-users",
});

// HTTPOnly cookies (if using cookies)
res.cookie("token", token, {
  httpOnly: true,
  secure: true,
  sameSite: "strict",
  maxAge: 24 * 60 * 60 * 1000,
});
```

#### 6. Cryptographic Attacks

**Key Disclosure**:

- Never log encryption keys
- Store keys separately from encrypted data
- Use environment variables in production

**Weak Algorithms**:

- Avoid: DES, MD5, SHA-1
- Use: AES-256, SHA-256, bcrypt

**Implementation Flaws**:

```javascript
// ❌ WRONG: Reusing IV
const iv = new Uint8Array(12); // Static IV

// ✅ CORRECT: Random IV each time
const iv = crypto.getRandomValues(new Uint8Array(12));
```
